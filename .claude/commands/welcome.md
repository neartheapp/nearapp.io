Welcome a new person to this project. Adapt your introduction based on what you find in the project's files.

Steps:
1. Read CLAUDE.md to understand the project
2. Scan the directory structure to understand how it's organized
3. Read PROJECT_LESSONS.md for any important lessons from past work

Then introduce the project in a friendly, plain-language way:

**Cover these topics:**
- What is this project? (1-2 sentences, no jargon)
- How is it organized? (list the main folders and what's in each)
- How do I run it? (the exact commands to get started)
- What's automated? (explain that hooks handle formatting, linting, type checking, and security automatically — they don't need to think about it)
- What commands are available? Explain /audit, /save, /share
- Any important lessons? (from PROJECT_LESSONS.md)
- What's the current state? Offer to run /audit to show project health

**End with:** "What would you like to work on?" so they can get started.

**Tone:** Friendly and encouraging, like helping a friend get set up. No jargon — explain technical terms in parentheses if needed. Follow the project's plain-language communication rule.
